Acoustic Pollution: Description back to ToC

Figure 1 shows classes and properties of the Acoustic Pollution ontology.

Figure 1 shows the ontology diagram which is organized in four modules representing subdomains related with acoustic pollution measurement. The module "Sensor" models the noise sensor noise:NoiseSensor which collects environmental noise measurements. The module "Measurement" models the properties representing the sensor measurements. The module "Time" allows the representation of time intervals and instants where the measurements were done. The module "Location" represents geographic location.

Figure 2 shows the ontology instances representing weightings, sensors classes, unit of measurement for the noise levels (decibels) and the types of acoustics emitters.

Example back to ToC

The following example shows the instances of: a sensor, an average noise level, and an observation. These instances together are the stage in which a sensor collects the measurement of the average noise level (Ld) captured during the month of April, 2017.

Example of instantiation of a noise sensor
ex:NoiseSensor01 rdf:type noise:NoiseSensor ;
                 ssn:observes ex:FeatureOfInterestPaseoDeRecoletos;
                 noise:belongsToSensorClass noise:ClaseI ;
                 noise:lastCalibrationDate ex:InstantCalibration20170330 ;
                 rdfs:label "NoiseSensor01"@es ;
                 wgs84_pos:location ex:NoiseSensor01Location .

The example describes a noise sensor linked by the property ssn:observes with the location of Paseo de Recoletos, where the measurement is captured. The class which the sensor belongs is represented by the property noise:belongsToSensorClass linked with the instance noise:ClaseI. The last calibration date of the sensor is represented by the property noise:lastCalibrationDate linked with the instance of a time instant ex:InstantCalibration20170330. Finally the geographic location, where the sensor is located, is represented by the instance ex:NoiseSensor01Location which has specific values for the geographical coordinates.

The instantiation of the geographic location of the sensor ex:NoiseSensor01Location and the time instant in which it was calibrated ex:InstantCalibration20170330 is performed as follows: 

ex:NoiseSensor01Location rdf:type wgs84_pos:SpatialThing ;
                         rdfs:label "Ubicación del sensor NoiseSensor01"@es ;
                         wgs84_pos:alt "648.0"^^xsd:float ;
                         wgs84_pos:lat "40.423332"^^xsd:float ;
                         wgs84_pos:long "-3.6908333"^^xsd:float .
					
ex:InstantCalibration20170330 rdf:type time:Instant ;
                              rdfs:comment "Instante de calibración"@es ;
                              time:inXSDDateTimeStamp "2017-03-30T14:25:20+01:00"^^xsd:dateTimeStamp .

Example of instantiation of an average noise level
ex:LdESP rdf:type noise:AverageNoiseLevel ;
         noise:hasAcousticEmitter "http://vocab.linkeddata.es/datosabiertos/kos/medio-ambiente/contaminacion-acustica/emisor-acustico/aglomeracion";
         noise:referenceInterval ex:IntervalLd ;
         rdfs:comment "Nivel de ruido día que se define como el valor medio del sonido medido en el intervalo de referencia 
	  de 7-19 horas según la normativa española"@es ;
         rdfs:label "Nivel de ruido día"@es ;
         rdfs:seeAlso "ISO 1996-2: 1987"@en .

The example describes the average noise level ex:LdESP corresponding to Ld (average noise level value in daytime) captured in the reference interval from 7 to 19h in Spain. The acoustic emitter, that produces this average noise level, is represented by the property noise:hasAcousticEmitter linked with the particular instance of the emitter noise:aglomeracion. The reference interval, when the measurement was performed, is represented by the property noise:referenceInterval linked with the instance of the time interval ex:IntervalLd.

The instantiation of the reference interval ex:IntervalLd for the average noise level ex:LdESP is performed as follows:

ex:IntervalLd rdf:type time:Interval ;
              time:hasBeginning ex:InstantBeginLd ;
              time:hasEnd ex:InstantEndLd ;
              rdfs:comment "Intervalo de referencia del nivel medio de ruido Ld en España, de 7:00 a 19:00."@es ;
              rdfs:label "Intervalo de referencia LdESP"@es .  

ex:InstantBeginLd rdf:type time:Instant ;
                  time:inDateTime ex:DateTimeDescriptionBeginLd ;
                  rdfs:comment "Inicio de medición del intervalo de ruido de dia"@es .			  
				  
ex:DateTimeDescriptionBeginLd rdf:type time:DateTimeDescription ;
                              time:unitType time:unitHour ;
                              time:hour "07"^^xsd:nonNegativeInteger ;
                              rdfs:comment "Descripción del inicio del instante del intervalo de medicion (07:00)"@es .

The code shows that the interval ex:IntervalLd has a start and end instant represented by the properties time:hasBeginning and time:hasEnd respectively. This properties are linked to the particular instances of time instants ex:InstantBeginLd and ex:InstantEndLd.

The instance of start instant ex:InstantBeginLd is linked to a description ex:DateTimeDescriptionBeginLd. This description allows to specify with the property time:inDateTime that the time instant is expressed in hours. The property time:inDateTime represents the position of an instant of time. Also the instance ex:DateTimeDescriptionBeginLd with the property time:unitType allows to represent the hour as the unit time of the interval, it is expresed by the particular instance time:unitHour. The property time:hour allows to define the particular hour, in this example the start time "07"^^xsd:nonNegativeInteger.

The instantiation of the end instant ex:InstantEndLd is similar to the explained about instantiation of ex:InstantBeginLd, but with the particular value for the end time (19h00).

Example of instantiation of an observation
ex:NoiseSensor01Observation_LdInterval201704 rdf:type ssn:Observation ;
                                             ssn:observedProperty ex:LdESP ;
                                             ssn:observedBy ex:NoiseSensor01 ;
                                             ssn:featureOfInterest ex:FeatureOfInterestPaseoDeRecoletos ;
                                             ssn:observationResult ex:NoiseSensor01SensorOutput_ObservationLdInterval201704 ;                                             
                                             ssn:observationSamplingTime ex:Interval201704 ;                                             
                                             rdfs:comment "Observación del nivel Ld realizada por el sensor NoiseSensor01 
					     durante el mes de abril de 2017"@es .

The example describes the instantiation of an observation. The property ssn:observedProperty represents the property that was observed, ex:LdESP in this example. The property ssn:observedBy represents sensor which performs the observation, ex:NoiseSensor01 in this example. The instances ex:NoiseSensor01 and ex:LdESP where explained in the previous examples.

This observation was performed for a feature of interest represented by the property ssn:featureOfInterest linked to the particular instance ex:FeatureOfInterestPaseoDeRecoletos, as previously was explained, which represents the location of Paseo del Prado. The sensor output, which represents the result of the observation, is indicated by the property ssn:observationResult linked with the instance ex:NoiseSensor01SensorOutput_ObservationLdInterval201704. Finally the interval, when the observation was performed, is represented by the property ssn:observationSamplingTime and is linked to the instance of time interval ex:Interval201704, which represents the month of april 2017.

The instance of the observation result, ex:NoiseSensor01SensorOutput_ObservationLdInterval201704, is represented as follows: 

ex:NoiseSensor01SensorOutput_ObservationLdInterval201704 rdf:type ssn:SensorOutput ;
                                                         ssn:hasValue ex:NoiseSensor01ObservationValue_ObservationLdInterval201704 ;
                                                         rdfs:comment "Salida del sensor NoiseSensor01 tras la observación del nivel Ld 
							 realizada durante el mes de abril de 2017"@es .

The observation result uses the property ssn:hasValue to link it with their particular value ex:NoiseSensor01ObservationValue_ObservationLdInterval201704, which is represented as follows:

ex:NoiseSensor01ObservationValue_ObservationLdInterval201704 rdf:type ssn:ObservationValue ;
                                                             noise:hasUnitOfMeasure "http://dbpedia.org/resource/Decibel" ;
                                                             noise:isWeightingWith noise:PonderacionA ;
                                                             rdf:value 67.3 ;
                                                             rdfs:comment "Valor de la observación del nivel Ld realizada por el sensor 
							     NoiseSensor01 durante el mes de abril de 2017"@es .

As you can see, the property noise:hasUnitOfMeasure is used to represent the unit of measure of the observation value. This property is linked to the decibel instance referenced to dbpedia:Decibel, that represents an instance of om:UnitOfMeasure. The weighting, used by sensor to perform the observation, is represented by the property noise:isWeightingWith linked to the observation value noise:PonderacionA, an instance of noise:Weighting. Finally the property rdf:value is used to represent the output of the observation value, "67.3"^^xsd:decimal in this example.
